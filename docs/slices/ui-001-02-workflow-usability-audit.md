# UI-001-02: Workflow and Usability Audit

## Status
Planning child of UI-001.

## Goal
Trace real user tasks and record where the current interface helps or obstructs completion.

## Dependencies
Requires UI-001-01.

## Scope
- Trace work-order creation/reopen, master-data management, assignment/scheduling, time review, parts review/purchasing, reports, administration, and technician progress capture.
- Record labels, navigation choices, context loss, hidden actions, scrolling, inconsistent controls, errors, and mobile constraints.
- Identify workflows and patterns that currently work well.
- Capture task-based screenshots at representative viewports.

## Required artifacts
- `docs/layout/current-state-workflow-map.md`
- Task observation notes with role, route, viewport, success/failure point, and evidence.
- Current-state workflow screenshot index.

## Acceptance criteria
- Each required task has a start point, path, outcome, and observed friction record.
- Findings distinguish workflow rules from presentation problems.
- No solution is treated as approved in this child.

## Guardrail
Do not prioritize or redesign findings here; consolidation belongs to UI-001-03.
