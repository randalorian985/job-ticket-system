# UI-000: User Interface Redesign Wireframe Plan

## Status
Planning parent scope only.

## Problem
The current interface is confusing, navigation does not scale cleanly, and several workflows require users to understand internal module structure before finding their next task.

## Goal
Produce evidence-based, reviewed wireframes and one approved UI specification through small planning children before broad production redesign begins.

## Required parent sequence
1. [UI-001 Current UI and Workflow Audit](ui-001-current-ui-workflow-audit.md)
2. [UI-002 Information Architecture and Application Shell](ui-002-information-architecture-and-application-shell-wireframes.md)
3. [UI-003 Work Order Workspace Wireframes](ui-003-work-order-workspace-wireframes.md)
4. [UI-004 Operations Workbench Wireframes](ui-004-operations-workbench-wireframes.md)
5. [UI-005 Technician Mobile Workflow](ui-005-technician-mobile-wireframes.md)
6. [UI-006 Validation and Approved UI Specification](ui-006-wireframe-validation-and-approved-ui-spec.md)

Each parent contains smaller planning children and is complete only when its children are reviewed.

## Shared principles
- Organize around user tasks and operational domains.
- Keep current work, ownership, status, blockers, and next action obvious.
- Reduce competing actions, oversized headers, decorative spacing, and unnecessary scrolling.
- Preserve every authorized capability and existing business rule.
- Use consistent page anatomy, queue, form, drawer, dialog, validation, and accessibility patterns.
- Design desktop, tablet, and mobile intentionally.
- Validate task completion and findability, not visual preference alone.

## Guardrails
- UI planning children do not change production components, CSS, routes, APIs, permissions, workflows, or data models.
- No parent is sent to Codex as one run.
- Broad production redesign remains blocked until UI-006-03 approval.
- Approval does not authorize one broad implementation; production work must be divided into focused implementation slices.

## Definition of done
All UI planning children are reviewed, the approved specification is documented, and a right-sized production UI implementation sequence exists.
